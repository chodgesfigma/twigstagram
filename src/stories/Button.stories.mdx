# Button Component

The `Button` component is a versatile UI element used across the Twigstagram application. It comes in various variants and sizes to suit different use cases.


## Usage

To use the `Button` component in your project, you can import it and use it with the desired props. Below is an example of how to include it in your application.

```jsx
import React from 'react';
import { Button } from '@/components/ui/button';

const App = () => (
  <>
    <Button variant="default" size="default">Default Button</Button>
    <Button variant="destructive" size="default">Destructive Button</Button>
    <Button variant="outline" size="default">Outline Button</Button>
    <Button variant="secondary" size="default">Secondary Button</Button>
    <Button variant="ghost" size="default">Ghost Button</Button>
    <Button variant="link" size="default">Link Button</Button>
    <Button variant="default" size="sm">Small Button</Button>
    <Button variant="default" size="lg">Large Button</Button>
    <Button variant="default" size="icon">Icon Button</Button>
  </>
);

export default App;
```

## Style Guidelines

When implementing the `Button` component, it is important to adhere to certain style guidelines to maintain a consistent look and feel throughout the application. Below are some recommended style guidelines:

### Consistency

- **Colors:** Use the defined color palette for all elements in the `Button`. Ensure that the colors match the overall theme of the application.
- **Typography:** Use the `Inter` font family, as specified in your global CSS. Apply appropriate text classes for consistent typography.

### Spacing and Alignment

- **Padding and Margin:** Maintain consistent padding and margin values. Use utility classes to ensure the `Button` looks balanced.
- **Alignment:** Ensure that all buttons are properly aligned, both horizontally and vertically.

### Responsiveness

- **Mobile-Friendly:** The `Button` should be fully responsive and work well on various screen sizes.
- **Viewport Width:** Make sure the `Button` spans the appropriate width for its context.

### Accessibility

- **Keyboard Navigation:** Ensure that the `Button` is accessible via keyboard navigation.
- **Aria Labels:** Use appropriate ARIA labels for the buttons to improve accessibility for screen readers.
- **Contrast:** Maintain sufficient contrast between the text and the background color to ensure readability.

### Interactions

- **Hover and Focus States:** Define clear hover and focus states for interactive elements. Use utility classes to style these states.
- **Active State:** Highlight the active or selected button to indicate the current action.

### Customization

- **Theming:** Allow for easy customization and theming of the `Button`. Ensure the component is styled using utility classes that can be easily adjusted.
- **Component Overrides:** Provide options to override specific styles or components within the `Button` to cater to unique use cases without compromising the overall design integrity.
