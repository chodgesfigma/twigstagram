# Input Component

The `Input` component is a versatile input field used in the Twigstagram application. It supports various types, including text, password, email, and number, and can be customized with placeholders and disabled states.

## Usage

To use the `Input` component in your project, you can import it and use it with the desired props. Below is an example of how to include it in your application.

```jsx
import * as React from "react"

import { cn } from "@/lib/utils"

export interface InputProps
  extends React.InputHTMLAttributes<HTMLInputElement> {}

const Input = React.forwardRef<HTMLInputElement, InputProps>(
  ({ className, type, ...props }, ref) => {
    return (
      <input
        type={type}
        className={cn(
          "flex h-10 w-full rounded-md border border-input bg-background px-3 py-2 text-sm ring-offset-background file:border-0 file:bg-transparent file:text-sm file:font-medium placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50",
          className
        )}
        ref={ref}
        {...props}
      />
    )
  }
)
Input.displayName = "Input"

export { Input }
```

## Style Guidelines

When implementing the `Input` component, it is important to adhere to certain style guidelines to maintain a consistent look and feel throughout the application. Below are some recommended style guidelines:

### Consistency

- **Colors:** Use the defined color palette for all elements in the `Input`. Ensure that the colors match the overall theme of the application.
- **Typography:** Use the `Inter` font family, as specified in your global CSS. Apply appropriate text classes for consistent typography.

### Spacing and Alignment

- **Padding and Margin:** Maintain consistent padding and margin values. Use utility classes to ensure the `Input` looks balanced.
- **Alignment:** Ensure that all elements within the `Input` are properly aligned, both horizontally and vertically.

### Responsiveness

- **Mobile-Friendly:** The `Input` should be fully responsive and work well on various screen sizes.
- **Viewport Width:** Make sure the `Input` spans the appropriate width for its context.

### Accessibility

- **Keyboard Navigation:** Ensure that the `Input` is accessible via keyboard navigation.
- **Aria Labels:** Use appropriate ARIA labels for the input fields to improve accessibility for screen readers.
- **Contrast:** Maintain sufficient contrast between the text and the background color to ensure readability.

### Interactions

- **Hover and Focus States:** Define clear hover and focus states for interactive elements. Use utility classes to style these states.
- **Active State:** Highlight the active or selected state to indicate the current action.

### Customization

- **Theming:** Allow for easy customization and theming of the `Input`. Ensure the component is styled using utility classes that can be easily adjusted.
- **Component Overrides:** Provide options to override specific styles or components within the `Input` to cater to unique use cases without compromising the overall design integrity.
